Spring Boot安全审计方法论：从攻击面映射到风险识别

官方定义：Spring Boot是由Pivotal团队提供的一套开源框架，可以简化spring应用的创建及部署。它提供了丰富的Spring模块化支持，可以帮助开发者更轻松快捷地构建出企业级应用。Spring Boot通过自动配置功能，降低了复杂性，同时支持基于JVM的多种开源框架，可以缩短开发时间，使开发更加简单和高效。

Spring boot框架

先来介绍一下spring boot是什么，SpringBoot是一个集成了Spring技术栈的一个大整合，是一个简化了Spring应用开发的框架。

特点

  • 创建独立的 Spring 应用
  • 直接嵌入 Tomcat、Jetty 或 Undertow（无需部署 WAR 文件）
  • 提供固执己见的“starter”依赖项，以简化您的构建配置
  • 尽可能自动配置 Spring 和第三方库
  • 提供生产级别的功能，例如指标、健康检查和外部化配置
  • 绝对没有代码生成，也无需 XML 配置

​ 单个的springboot项目，为单体应用。单体应用就是指把所有的应用需要的资源放到一个项目里面，即所有的功能都集中在单进程里面，那么可能由于我们的服务器运载能力较差我们就会用多个服务器来跑这一个项目，那么这是就需要把整个项目复制到另外的服务器上。这就是单体应用。

微服务

​ 微服务（Microservices）是一种软件架构风格，它将一个大型的单体应用（Monolithic Application）拆分成一组小型、独立、松耦合的服务，每个服务围绕特定的业务功能构建，可以独立开发、部署、扩展和运维。核心思想，分而治之” + “单一职责”

​ 微服务:我们将各个功能模块分割出来为单独的功能元素，那么每个功能放在独立的服务上，那么每一个功能由于独立出来我们在后面进行跨服务器分发这些服务的时候就可以有选择性组合(各功能的组合是通过http来实现的互联)，而不需要像单体应用那样复制所有的功能。

二者联系，Spring Boot 是实现单体应用和微服务的通用技术基础，而微服务是一种架构思想——Spring Boot 既可以用于构建单体应用，也可以作为微服务的基本单元。

后端代码文件夹

审计角度

开始看后端代码，首先看filter过滤器，首先看@WebFilter注解，过滤器器的名：adminPermissionFilter；要过滤的拦截路径：/admin/*，后端代码拦截url中带有/admin的路径。

每个Fileter都是继承javax.servlet.Fileter, 每个函数前@Override都有注解,重写（Override）是指子类定义了一个与其父类中具有相同名称、参数列表和返回类型的方法，并且子类方法的实现覆盖了父类方法的实现。即外壳不变，核心重写！

执行顺序：1、init() 初始化方法，在 Web 应用启动时调用一次；2、doFilter()核心方法，每次请求都会经过此方法；3、estroy() 销毁方法，在应用关闭时调用。

WebFilter是 Java Servlet 标准中的一个接口，用于在请求到达 Controller 之前或响应返回客户端之前执行一些通用操作。

在访问http://localhost:8080/tmall/admin/home ，先走到Filter过滤器，判断路径中是否有/admin/login或/admin/account后判断用户权限，当adminId为空则跳转，成功则走到goToPageByAjax()函数，来执行具体逻辑。

分析权限是由adminId，没有判断cookie或者jwt来区分，查看登录逻辑，传入的username和password查询用户，查询的封装admin类，Admin类中有getAdmin_id，根据getAdmin的值来判断是否为管理员。

getAdmin_id的值为后端服务进行查询，无法通过前端进行修改，无法绕过。

重点审计面

Controller 层

查找 @RequestMapping, @PostMapping, @GetMapping 等注解暴露的接口。

重点检查：1、是否有未授权访问接口（无 @PreAuthorize、无 session 判断）；2、是否存在未过滤的文件操作接口（如 /upload, /download）；3、是否存在拼接 SQL 或反射调用的逻辑

Filter / Interceptor 层

Filter控制访问范围，但可能存在绕过逻辑（URL大小写、路径混淆、静态资源绕过）。

Interceptor常用于认证与鉴权逻辑，重点关注：1、登录状态如何保存（Session / Cookie / JWT）；2、是否存在 “白名单路径过宽”；3、是否可通过伪造请求头绕过（如 X-Forwarded-For 判断错误）

DAO 层（MyBatis）

重点：检查 Mapper.xml 中是否存在 ${} 拼接（未预编译）。

审计链路分析：

  1. 识别防护边界 - 找到权限校验的入口(@WebFilter("/admin/*"))
  2. 理清校验逻辑 - 分析doFilter中的权限判断流程
  3. 寻找绕过的可能性 - 检查是否有路径白名单、权限校验是否统一
  4. 追踪权限标识 - 分析adminId的生成、存储、验证机制
