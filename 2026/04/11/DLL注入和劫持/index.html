shellcode loader

常规的loader有五种。1、指针执行；2、申请动态内存；3、嵌入汇编4；、强制类型转换；5、汇编花指令。这五种方式都是通过编译打包成一段exe程序，双击exe程序直接进行执行。运行在任务管理器中会多一个进程，杀毒软件对独立的 .exe 文件监控最严，独立的 .exe 第一次联网时，Windows 防火墙通常会弹出警告询问用户，缺点特别明显，极其容易暴露。

DLL概念和用途

dll(dynamic-link library),动态链接库，是微软实现共享函数库的一种方式。动态链接，就是把一些常用的函数代码制作成dll文件，当某个程序调用到dll中的某个函数的时候，windows系统才把dll加载到内存中。也就是说当程序需要的时候才链接dll，所以是动态链接。

用途：动态链接的好处是磁盘和内存的消耗减少，这也是dll的最初目的。同时dll也提供了模块化的函数库。

EXE和DLL

EXE 和 DLL 都是 PE 文件，它们的区别在于文件头中的标记位。DLL 被调用的模式主要分为 静态加载 和 动态加载。

静态加载(利用点劫持)

这种模式发生在 进程初始化阶段。机制原理，当一个 EXE 启动时，Windows 的 PE 加载器 会首先读取 EXE 文件头中的 引入表。引入表明确列出了该程序依赖的所有 DLL 名称（如 kernel32.dll, user32.dll）以及需要调用的函数名称或序号。

静态执行流程
  1. 映射文件：加载器将 EXE 映射到内存。
  2. 解析依赖：加载器扫描引入表，发现缺失的 DLL。
  3. 加载 DLL：加载器在系统目录中找到这些 DLL，将它们映射到进程的虚拟地址空间。
  4. 地址修正：加载器将 DLL 中函数的实际内存地址填入 EXE 的 引入地址表 中。
  5. 启动执行：所有依赖的 DLL 加载完毕后，EXE 的入口点才被执行。

动态加载模式(利用点注入)

这种模式发生在 进程运行阶段。机制原理，程序在运行时，通过调用 Windows API 显式地请求操作系统加载 DLL。此时，DLL 的文件路径是一个字符串变量，而不是写在文件头的引入表里。

动态执行流程
  1. 请求加载：程序执行到特定逻辑时，调用 LoadLibrary 或 LoadLibraryEx。
  2. 内存映射：操作系统内核将该 DLL 映射到当前进程的地址空间（如果该 DLL 尚未加载）。
  3. 获取地址：程序调用 GetProcAddress，传入 DLL 的句柄和函数名。系统会在 DLL 的 导出表 中查找该函数的内存地址并返回。
  4. 函数调用：程序通过函数指针调用该地址。
  5. 卸载：使用完毕后，调用 FreeLibrary 减少引用计数，从内存中卸载 DLL。

DLL优点

DLL会寄生在已经运行的一个程序中，不会另外新起一个进程。如果 DLL 注入到了一个已经被防火墙信任的进程（比如浏览器或系统组件）中，它发出的网络流量通常会被防火墙视为该合法进程的流量，从而直接放行，不会弹窗报警。

DLL注入(动态加载)

让程序加载一个DLL，有LoadLibraryW函数。

LoadLibraryW将指定的模块加载到调用进程的地址空间中。指定的模块可能会加载其他模块。

注入思路

1、打开进程获取句柄；2、申请shell coder空间；3、写入内存；4、获取LoadLibraryW；5、创建线程；6、等待线程结束；7、释放dll；8、关闭进程句柄
