PE文件五

PE文件结构大概分为Dos头、Nt头、节表、引入表、资源表、重定位表、调试表。除节表文件以外，其他的都是记录这个程序的启动或运行参数。

节表

节表中存储的是对具体内容的存储，常见的有：.text、.data、.rdata、.idata、.edata、.rsrc、.reloc、.bss。

.text节 (代码段)

-存放内容：程序的核心可执行机器码（Machine Code）、常量字符串（如 printf中的提示语）以及编译器生成的辅助函数等。
-典型属性：可读、可执行（不可写）。

.data 节 (已初始化数据段)

-存放内容：程序中已经赋予了初始值的全局变量和静态变量。例如 C++ 代码中 int globalVar = 10; 这个变量及其初始值就存在这里。
典型属性：可读、可写。

.rdata 节 (只读数据段 / Read-Only Data)

放内容：程序运行时需要读取但不允许修改的数据。包括导入表（Import Table）的函数名称、常量数组、虚函数表（vtable）等。
典型属性：仅可读。

.idata 节 (导入表 / Import Directory)

存放内容：程序依赖的外部 DLL 模块及其导出的 API 函数信息。比如你的程序调用了 MessageBoxA，这个函数的名字和所在的 DLL (user32.dll) 就记录在这里。
典型属性：读取、写入。

.edata 节 (导出表 / Export Directory)

存放内容：当前 PE 文件提供给其他程序使用的函数列表。
典型属性：读取。

.rsrc 节 (资源段 / Resources)

存放内容：程序的图形界面元素和非代码素材。比如图标（Icon）、对话框模板、菜单、位图（Bitmap）、版本信息、甚至嵌入的其他二进制文件。
典型属性：读取。

.reloc 节 (重定位表 / Relocations)

存放内容：记录了代码或数据中所有使用了“硬编码绝对地址”的位置。
典型属性：读取。

.bss 节 (未初始化数据段)

存放内容：声明了但没有赋予初始值的全局/静态变量（如 int buffer[1024];）。
典型属性：读取、写入。

未能正常运行

按照思路应该是先运行B程序，再运行A程序，运行A_merged.exe，发现程序B和程序A都没有运行。

查看A_merged程序结构，入口点地址是injct节，并且injct节已经添加了

原因一

1、B 的代码在编译时，所有绝对地址引用（比如全局变量、静态变量、虚函数表 等）都是基于 B 原来的基址（ImageBase）计算的。

2、B程序调用函数时用到的导入表，同样未能进行正确的查找，导致B程序无法正常运行。

思路转变

把B程序直接使用工具转化成一段shellcode代码，转化以后变成bin文件，bin文件直接添加到一个节中，并且把程序入口直接改成改节的RVA地址。

程序运行

发现只运行了B程序未能运行A程序，B的弹窗已经正常弹出，A程序的计算器未能弹出。

原因二

通过调试发现，B程序在程序结束后会使用NtTerminateProcess函数结束当前进程。B程序原本 的入口点是 mainCRTStartup 这类 CRT 函数，它执行完后会调用 exit() 或 ExitProcess()，根本不会顺序执行到你追加的那条 E9 跳转，最终整个进程也会在 B 的 main 返回后直接终止，宿主 A 的原始逻辑永远不会被执行。（解决办法是用纯汇编写一段代码，里面不包含退出函数）。
