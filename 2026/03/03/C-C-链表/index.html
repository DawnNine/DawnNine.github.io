C/C++链表

链表是物理存储单元上非连续的、非顺序的存储结构，解决了非连续空间利用问题，常见的数组字符串都是要连续一块内存进行存储，链表系统内存碎片化，空间得不到利用的问题。

​ 链表分为单向链表和双向链表，在创建单项链表时只会声明一个指针用来保存下一个值的地址，不保存上一个地址。在创建双线链表时会声明两个指针，一个是用来保存上一个数据的地址，一个是用来保存下一个数据的地址。

单向链表

​ 单项链表有头插法和尾插法，头插法是从头部进行插入数据，尾插法是在末尾插入数据。

头插法

​ 现在有一个链表，里面的数据有A、B、C。我现在想把D这个数据插入到A和B之间，要先把D.next=A.next，再把A.next=D

这个就是从头部一个一个插入。

尾插法

​ tmp指针永远指向最后一个链表数据，当添加一个元素，就要把tmp的指针放到最后一个元素上。

双向链表

​ 每添加一个节点，需要把前一个指针next执行添加的节点，添加的指针节点prior指向当前节点。

双向链表插值

目前是想把NodeA增加到中间其中两个值中，例如2和3的中间要插入一个NodeA。

1、首先判断当前值是否为2，不是接着循环，直到cur==NULL

if(cur.data==2)

2、声明一个NodeA指针，如果进入循环，NodeA.next同样指向cur.next的地址，NodeA.prev指向cur。

Node* NodeA = (Node*)malloc(sizeof(Node));
NodeA->data = 20; // 新节点的值
NodeA->next = cur->next;
NodeA->prev=cur;

3、进行cur.next指向NodeA地址

cur->next=NodeA

对链表进行循环找，直到链表为空。

总结

​ 链表比较抽象，它解决了数组在动态性方面的限制问题——让数据元素可以在不需要连续内存的情况下，动态插入、删除、扩展。
